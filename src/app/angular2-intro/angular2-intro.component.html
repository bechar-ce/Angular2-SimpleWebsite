Angular 2 Introduction

Speed & Performance, modular and mobility driven

Angular 2 is entirely component based. Controllers and $scope are no longer used in Angular 2. They have been replaced by components and directives. Components are directives with a template. AtScript is a language that is a superset of ES6 and it's being used to author Angular 2.0. It uses TypeScript's type syntax to represent optional types which can be used to generate runtime type assertions, rather than compile-time checks.

Why Angular 2?

  • Angular 2 is 5 times faster as compared to Angular 1.x
  • Angular 2 is mobile oriented & better in performance and speed
  • Angular 2 provides more choice for languages (Type Scrip, etc.)
  • Angular 2 implements web standards like components
  • Angular 2 is not that easy to setup as AngularJS 1.x
  • Angular 2 controllers and $scope has gone
  • Angular 2 has different ways to define local variables
  • Angular 2 has structural directives syntax is changed
  • Angular 2 uses camelCase syntax for built-in directives
  • Angular 2 directly uses the valid HTML DOM element properties and events
  • Angular 2 has One-way data binding directive replaced with [property]
  • Angular 2 has Two-way data binding: ng-model replaced with [(ngModel)]
  • Angular 2 has way of Bootstrapping Angular Application is changed
  • Angular 2 has ways of Dependency Injection is Changed- syntax changed
  • Angular 2 has way of routing is Changed- syntax changed
  • Angular 2 has very powerful routes
  • Angular 2 Router will only load components when it absolutely needs them

Same Code Sharing Between Web & Mobile Apps

Compare to Angular 1.x it was not built with mobile support but Angular 2 is mobile oriented and fully supported. Angular 2 is using Hierarchical Dependency Injection system which is major performance booster. Angular 2 implements unidirectional tree based change detection which again increases performance. As per ng-conf meetup, angular 2 is 5 times faster as compared to angular 1.

What are the advantages and disadvantages of using Angular2

Advantages (Pros)

  • The Testability of the code will be improved by decoupling DOM manipulation from application logic in Angular 2
  • In Angular 2 client side will be decoupled from the server side, so development work will progress in parallel. It will improved productivity
  • In Angular 2 the amount of JavaScript coding will be reduced
  • In Angular 2 no need to programmatically manipulate the DOM
  • In Angular 2 code will not describe the implementation details, but only the overall flow of the application
  • In Angular 2 will auto-inject services into the application and you will be able to develop features quickly in Application. Google promises that you will get full control over the initialization process in automated tests

Disadvantages (Cons)

  • It's complete different coding styles and and until Developer get used to Angular 2, they will struggle a little to understand how to integrate different components
  • Developers have different coding styles so need to learn the code
  • In Angular 2 you can do the same thing with multiple ways
